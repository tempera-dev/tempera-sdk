> **Private access.** Hosted Tempera services are available through design-partner onboarding.
> Use the issuer, credentials, environment, and product URLs provisioned for your workspace;
> start in staging unless Tempera explicitly approves another target.

The unified MCP gateway lives at **`${issuer}/mcp`**, using the issuer URL
supplied during onboarding. It uses stable MCP `2025-11-25` over
session-aware Streamable HTTP.
It requires a bearer minted for audience **`tempera-mcp`** with scope
**`mcp:invoke`** â€” or a central `tp_` API key.

The gateway aggregates every product MCP server behind **namespaced tools**
(`palette_*`, `tempo_*`, `cradle_*`, `remi_*`, `data_engine_*`), so one endpoint and one
credential can reach the whole fleet. Progressive mode keeps product schemas behind
discovery builtins; hybrid and full modes expose selected or complete namespaced catalogs.
Calls are metered as
`mcp_invocations` against the workspace's plan.
The SDK clients complete initialization automatically, preserve session and protocol
headers, accept JSON or SSE responses, bound pagination, and recover one expired session.

## Builtin tools

Alongside the namespaced product tools, the gateway serves its own builtins
(not metered as product invocations):

- **`tempera_search_tools`** â€” Search compact tool summaries without loading the complete schema catalog.
- **`tempera_describe_tool`** â€” Fetch one discovered tool's complete schema and surface hash.
- **`tempera_call`** â€” Invoke one progressively discovered namespaced tool.
- **`tempera_whoami`** â€” Fetch the caller's identity, workspace, and scopes as seen by the gateway.
- **`tempera_status`** â€” Fetch gateway upstream health for every connected product MCP server.

## Plan-limit errors

When the workspace exhausts its metered `mcp_invocations` allowance, tool
calls fail with JSON-RPC error code `-32002`
(`plan_limit_exceeded`). See the full [error-code table](/errors#mcp-gateway-error-codes).

## Client examples

All three packages ship a session-aware `TemperaMcpClient`. The Rust client
delegates wire behavior to the pinned official MCP Rust SDK; `McpRequestBuilder`
remains available for HTTP-less integrations.

<CodeGroup>

```typescript TypeScript
import { TemperaAuth, TemperaMcpClient } from "@tempera/sdk";

const issuerUrl = process.env.TEMPERA_ISSUER_URL;
if (!issuerUrl) throw new Error("Missing TEMPERA_ISSUER_URL");
const auth = new TemperaAuth({ issuerUrl, apiKey: process.env.TEMPERA_API_KEY });
const mcp = new TemperaMcpClient({ auth }); // url derives as ${issuer}/mcp

const tools = await mcp.listTools(); // initializes automatically
const matches = await mcp.searchTools("cradle capabilities");
const result = await mcp.callDiscoveredTool("cradle_get_capabilities");
console.log(await mcp.whoami());
console.log(await mcp.status());
await mcp.close();
```

```python Python
import os
from tempera_sdk import TemperaAuth, TemperaMcpClient

auth = TemperaAuth(issuer_url=os.environ["TEMPERA_ISSUER_URL"], api_key=os.environ["TEMPERA_API_KEY"])
mcp = TemperaMcpClient(auth=auth)  # url derives as ${issuer}/mcp

tools = mcp.list_tools()  # initializes automatically
matches = mcp.search_tools("cradle capabilities")
result = mcp.call_discovered_tool("cradle_get_capabilities")
print(mcp.whoami())
print(mcp.status())
mcp.close()
```

```rust Rust
use tempera_sdk::{TemperaMcpClient, TemperaAuth};

let issuer_url = std::env::var("TEMPERA_ISSUER_URL")?;
let api_key = std::env::var("TEMPERA_API_KEY")?;
let auth = TemperaAuth::new(issuer_url).with_api_key(api_key);
let bearer = auth.bearer_for("tempera-mcp").expect("MCP credential");
let mcp = TemperaMcpClient::connect(auth.mcp_url(), bearer).await?;
let tools = mcp.list_tools().await?;
let matches = mcp.search_tools("cradle capabilities", None, Some(5), false).await?;
let result = mcp.call_discovered_tool("cradle_get_capabilities", Default::default()).await?;
let identity = mcp.whoami().await?;
mcp.close().await?;
```
