> **Private access.** Hosted Tempera services are available through design-partner onboarding.
> Use the issuer, credentials, environment, and product URLs provisioned for your workspace;
> start in staging unless Tempera explicitly approves another target.

The unified MCP gateway lives at **`${issuer}/mcp`**, using the issuer URL
supplied during onboarding. It is a stateless streamable-HTTP JSON-RPC 2.0 endpoint.
It requires a bearer minted for audience **`tempera-mcp`** with scope
**`mcp:invoke`** — or a central `tp_` API key.

The model-facing surface is a fixed **capability fabric**, so product
capabilities are not flat namespaced tools. Use `tempera_search` to discover
policy-filtered opaque cards, `tempera_describe` to inspect one, then
`tempera_invoke` or `tempera_prepare`/`tempera_commit` to execute it. Product
execution is metered as
`mcp_invocations` against the workspace's plan.

## JSON-RPC methods

| Method | JSON-RPC | Tool | Description |
|---|---|---|---|
| `initialize` | `initialize` | — | Open an MCP session and fetch server capabilities and instructions. |
| `ping` | `ping` | — | Check gateway liveness over JSON-RPC. |
| `listTools` | `tools/list` | — | List the fixed ten-verb Tempera capability-fabric surface; product cards never appear as flat product tool names. |
| `callTool` | `tools/call` | — | Call a fixed fabric verb. Discover product cards with tempera_search, inspect a card with tempera_describe, then invoke its opaque capability reference through tempera_invoke or tempera_prepare/tempera_commit. |
| `whoami` | `tools/call` | `tempera_whoami` | Fetch the caller's identity, workspace, and scopes as seen by the gateway. |
| `status` | `tools/call` | `tempera_status` | Fetch gateway upstream health for every connected product MCP server. |

## Builtin tools

The fixed surface includes these gateway verbs (product-card discovery itself
is not a product invocation):

- **`tempera_whoami`** — Fetch the caller's identity, workspace, and scopes as seen by the gateway.
- **`tempera_status`** — Fetch gateway upstream health for every connected product MCP server.

## Plan-limit errors

When the workspace exhausts its metered `mcp_invocations` allowance, tool
calls fail with JSON-RPC error code `-32002`
(`plan_limit_exceeded`). See the full [error-code table](/errors#mcp-gateway-error-codes).

## Client examples

TypeScript and Python ship a `TemperaMcpClient`; the HTTP-less Rust crate
ships `McpRequestBuilder`, which produces the exact JSON-RPC bodies to POST
at `TemperaAuth::mcp_url()`.

<CodeGroup>

```typescript TypeScript
import { TemperaAuth, TemperaMcpClient } from "@tempera/sdk";

const issuerUrl = process.env.TEMPERA_ISSUER_URL;
if (!issuerUrl) throw new Error("Missing TEMPERA_ISSUER_URL");
const auth = new TemperaAuth({ issuerUrl, apiKey: process.env.TEMPERA_API_KEY });
const mcp = new TemperaMcpClient({ auth }); // url derives as ${issuer}/mcp

await mcp.initialize();
const tools = await mcp.listTools();
const cards = await mcp.callTool("tempera_search", { query: "browser capability" });
console.log(await mcp.whoami());
console.log(await mcp.status());
```

```python Python
import os
from tempera_sdk import TemperaAuth, TemperaMcpClient

auth = TemperaAuth(issuer_url=os.environ["TEMPERA_ISSUER_URL"], api_key=os.environ["TEMPERA_API_KEY"])
mcp = TemperaMcpClient(auth=auth)  # url derives as ${issuer}/mcp

mcp.initialize()
tools = mcp.list_tools()
cards = mcp.call_tool("tempera_search", {"query": "browser capability"})
print(mcp.whoami())
print(mcp.status())
```

```rust Rust
use tempera_sdk::{McpRequestBuilder, TemperaAuth, parse_mcp_error};

let issuer_url = std::env::var("TEMPERA_ISSUER_URL")?;
let auth = TemperaAuth::new(issuer_url).with_api_key(api_key);
let mut mcp = McpRequestBuilder::new();

// POST each body at auth.mcp_url() with the tempera-mcp bearer:
let (id, body) = mcp.initialize_body("tempera-sdk", "0.9.0");
let (id, body) = mcp.list_tools_body();
let (id, body) = mcp.call_tool_body("tempera_search", Some(serde_json::json!({"query": "browser capability"})));
let (id, body) = mcp.whoami_body();

// Feed error responses to the shared parser:
if let Some(err) = parse_mcp_error(&response_body) {
    eprintln!("MCP error {}: {}", err.code, err.message);
}
```
